
Анализ предметной области проектной организации
Предметная область охватывает деятельность проектной организации, включая взаимодействие с заказчиками, распределение работы по отделам и сотрудникам, а также управление проектами.
Основные сущности предметной области:
Отделы: Представляют структурные подразделения организации, ответственные за выполнение проектных работ.
Сотрудники: Включают персонал организации, задействованный в проектной деятельности.
Организации: Представляют заказчиков проектных работ.
Договора: Фиксируют соглашения между проектной организацией и заказчиками на выполнение работ.
Проектные работы: Описывают конкретные проекты, выполняемые организацией по договорам с заказчиками.
Связи между сущностями:
Отделы - Сотрудники: Отдел имеет множество сотрудников, сотрудник принадлежит к одному отделу (one-to-many).
Сотрудники - Проектные работы: Сотрудник может участвовать в нескольких проектах, проект может выполняться несколькими сотрудниками (many-to-many).
Организации - Договора: Организация может заключать несколько договоров, договор заключается с одной организацией (one-to-many).
Договора - Проектные работы: Договор может включать несколько проектных работ, проектная работа относится к одному договору (one-to-many).
Отделы - Проектные работы: Отдел может выполнять несколько проектных работ, проектная работа выполняется одним отделом (one-to-many).
Первичные и внешние ключи:
Отделы:
Первичный ключ: ID_отдела
Внешний ключ: ID_начальника_отдела (ссылка на таблицу "Сотрудники")
Сотрудники:
Первичный ключ: ID_сотрудника
Внешний ключ: ID_отдела (ссылка на таблицу "Отделы")
Организации:
Первичный ключ: ID_организации
Договора:
Первичный ключ: ID_договора
Внешний ключ: ID_организации (ссылка на таблицу "Организации")
Проектные работы:
Первичный ключ: ID_проекта
Внешний ключ: ID_договора (ссылка на таблицу "Договора")
Внешний ключ: ID_отдела (ссылка на таблицу "Отделы")
Инфологическая модель базы данных:
Инфологическая модель представляет собой ER-диаграмму, которая отображает сущности, атрибуты и связи между ними, как описано выше.
Даталогическая модель базы данных:
Даталогическая модель представляет собой схему реляционной базы данных с таблицами, столбцами и типами данных.
Таблица "Отделы":
Столбец	Тип данных	Описание
ID_отдела	INTEGER	Первичный ключ, уникальный идентификатор отдела
Название отдела	VARCHAR(255)	Название отдела
Этаж	INTEGER	Номер этажа, на котором расположен отдел
Телефон	VARCHAR(20)	Номер телефона отдела
ID_начальника_отдела	INTEGER	Внешний ключ, ссылка на ID сотрудника (начальник отдела)
Таблица "Сотрудники":
Столбец	Тип данных	Описание
ID_сотрудника	INTEGER	Первичный ключ, уникальный идентификатор сотрудника
ФИО	VARCHAR(255)	Фамилия, имя и отчество сотрудника
Должность	VARCHAR(255)	Должность сотрудника (конструктор, инженер и т.д.)
ID_отдела	INTEGER	Внешний ключ, ссылка на ID отдела, в котором работает сотрудник
Пол	VARCHAR(10)	Пол сотрудника (мужской, женский)
Адрес	VARCHAR(255)	Адрес проживания сотрудника
Дата рождения	DATE	Дата рождения сотрудника

Эта даталогическая модель предоставляет структуру для хранения данных проектной организации и позволяет эффективно управлять информацией о отделах, сотрудниках, организациях, договорах и проектных работах.
